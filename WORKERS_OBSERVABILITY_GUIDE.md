# Cloudflare Workers 관찰 가능성 탭 가이드

## 📊 각 탭의 용도

### 1. **이벤트 (Events)** - 가장 중요! ⭐

**용도**: 개별 요청의 상세 로그 확인

**확인할 내용**:
- ✅ 각 요청의 타임스탬프
- ✅ CPU 시간과 실제 실행 시간
- ✅ **오류 메시지와 스택 트레이스** (가장 중요!)
- ✅ 요청/응답 데이터

**사용 방법**:
1. "이벤트" 탭 클릭
2. 테이블에서 특정 이벤트 클릭
3. 상세 로그 확인:
   - 오류 메시지
   - 스택 트레이스
   - 요청/응답 데이터

**언제 사용**:
- ❌ 오류가 발생했을 때
- 🔍 특정 요청이 실패한 이유를 알고 싶을 때
- 📝 상세한 로그를 확인하고 싶을 때

---

### 2. **호출 (Calls)**

**용도**: API 호출의 전체 흐름과 성능 분석

**확인할 내용**:
- ✅ API 호출 성공/실패 통계
- ✅ 평균 응답 시간
- ✅ 호출 그래프
- ✅ 에러율

**언제 사용**:
- 📈 전체 성능을 모니터링하고 싶을 때
- 📊 통계를 확인하고 싶을 때
- 🔍 패턴을 분석하고 싶을 때

---

### 3. **추적 (Tracing)**

**용도**: 분산 추적 정보 (고급)

**확인할 내용**:
- ✅ 요청의 전체 실행 흐름
- ✅ 각 단계별 소요 시간
- ✅ 외부 API 호출 추적

**언제 사용**:
- 🔍 복잡한 요청의 실행 흐름을 추적하고 싶을 때
- ⏱️ 성능 병목을 찾고 싶을 때

---

### 4. **쿼리 (Query)**

**용도**: D1 데이터베이스 쿼리 성능 분석

**확인할 내용**:
- ✅ D1 쿼리 실행 시간
- ✅ 쿼리 통계
- ✅ 느린 쿼리 식별

**언제 사용**:
- 🗄️ 데이터베이스 성능을 확인하고 싶을 때
- 🐌 느린 쿼리를 찾고 싶을 때

---

## 🎯 현재 상황에서 추천

### 오류 디버깅 중이라면:

1. **"이벤트" 탭** 클릭 ⭐
2. 최근 이벤트 중 **빨간색(오류) 표시된 이벤트** 클릭
3. 상세 로그 확인:
   - 오류 메시지
   - 스택 트레이스
   - 요청 데이터

### 성능 확인 중이라면:

1. **"호출" 탭** 클릭
2. 성공/실패 통계 확인
3. 평균 응답 시간 확인

### 데이터베이스 성능 확인 중이라면:

1. **"쿼리" 탭** 클릭
2. D1 쿼리 실행 시간 확인
3. 느린 쿼리 식별

---

## 📋 체크리스트

### 오류가 발생했을 때:

- [ ] "이벤트" 탭에서 최근 이벤트 확인
- [ ] 빨간색(오류) 이벤트 클릭
- [ ] 오류 메시지 복사
- [ ] 스택 트레이스 확인
- [ ] 오류 메시지를 개발자에게 전달

### 성능 문제가 있을 때:

- [ ] "호출" 탭에서 평균 응답 시간 확인
- [ ] "쿼리" 탭에서 D1 쿼리 시간 확인
- [ ] "추적" 탭에서 병목 지점 확인

---

## 💡 팁

1. **"이벤트" 탭이 가장 유용합니다!**
   - 오류 디버깅에 필수
   - 상세한 로그 확인 가능

2. **필터 사용**:
   - "Add filter" 버튼으로 특정 조건 필터링
   - 예: 오류만 보기, 특정 시간대만 보기

3. **실시간 모니터링**:
   - "Live" 버튼 클릭하여 실시간 로그 확인

4. **검색 기능**:
   - "전체 텍스트 로그 검색"으로 특정 키워드 검색

---

**현재 상황에서는 "이벤트" 탭에서 오류가 있는 이벤트를 클릭하여 상세 로그를 확인하는 것이 가장 중요합니다!**
