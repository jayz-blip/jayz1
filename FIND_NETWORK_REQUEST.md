# Network 탭에서 요청 찾기 가이드

## 🔍 Network 탭에서 요청 찾는 방법

### 1단계: Network 탭 열기

1. **F12** 키 누르기
2. **Network** 탭 클릭
3. **필터 초기화**: "All" 선택 (필터가 켜져 있으면 해제)

### 2단계: 요청 찾기

1. **채팅 메시지 전송** (예: "안녕하세요")
2. **Network 탭에서 확인**:
   - **XHR** 또는 **Fetch** 필터 클릭
   - 또는 **All**에서 스크롤하여 찾기
   - 요청 이름: `chat` 또는 `api/chat`

### 3단계: 요청 정보 확인

요청을 클릭하면:

1. **Headers 탭**:
   - **Request URL**: 요청 URL 확인
   - **Request Method**: POST 확인
   - **Status Code**: 상태 코드 확인

2. **Response 탭**:
   - 응답 내용 확인
   - 오류 메시지 확인

## 🔍 요청이 보이지 않는 경우

### 경우 1: 요청이 전송되지 않음

**증상**: Network 탭에 아무 요청도 없음

**원인**: 
- JavaScript 오류로 요청이 차단됨
- 네트워크 연결 문제

**해결**:
1. **Console 탭**에서 빨간색 오류 확인
2. 오류 메시지 복사

### 경우 2: 필터링됨

**증상**: 요청이 있지만 보이지 않음

**해결**:
1. **Network 탭**에서 필터 확인
2. **"All"** 선택
3. **검색창** 비우기

### 경우 3: 요청이 실패함

**증상**: 요청이 보이지만 빨간색으로 표시됨

**확인 사항**:
1. **상태 코드** 확인 (404, 500, CORS 등)
2. **Response 탭**에서 오류 메시지 확인

## 📋 확인할 정보

요청을 찾으면 다음 정보를 확인하세요:

1. **요청 URL**: 
   - ✅ 올바름: `/api/chat` 또는 `https://jayz1.pages.dev/api/chat`
   - ❌ 잘못됨: `http://localhost:8000/api/chat`

2. **상태 코드**:
   - ✅ 200: 성공
   - ❌ 404: 경로를 찾을 수 없음
   - ❌ 500: 서버 오류
   - ❌ CORS: CORS 오류

3. **Response 내용**:
   - 오류 메시지 확인

## 🚀 빠른 확인 방법

### 방법 1: XHR 필터 사용

1. **Network 탭** → **XHR** 클릭
2. **채팅 메시지 전송**
3. **요청 확인**

### 방법 2: 검색 사용

1. **Network 탭** → **검색 아이콘** 클릭
2. **"chat"** 또는 **"api"** 입력
3. **요청 찾기**

### 방법 3: Preserve log 사용

1. **Network 탭** → **Preserve log** 체크
2. **페이지 새로고침**
3. **채팅 메시지 전송**
4. **요청 확인**

**요청을 찾으면 상태 코드와 Response 내용을 알려주세요!**
