# Workers 로그 찾기 가이드

## 🔍 Workers 로그 위치

Workers 대시보드에서 로그는 **"관찰 가능성"** (Observability) 탭에 있습니다.

## 📋 탭 설명

1. **개요** (Overview): Workers 기본 정보
2. **메트릭** (Metrics): 성능 지표
3. **배포** (Deployments): 배포 내역
4. **바인딩** (Bindings): D1, AI 등 바인딩 설정
5. **관찰 가능성** (Observability): **로그 및 추적 정보** ⭐
6. **설정** (Settings): Workers 설정

## 🚀 로그 확인 방법

### 1단계: 관찰 가능성 탭 클릭

1. **Workers 대시보드** → **`chatbot-api` 프로젝트**
2. **"관찰 가능성"** (Observability) 탭 클릭

### 2단계: 로그 확인

"관찰 가능성" 탭에서 다음을 확인할 수 있습니다:
- **실시간 로그** (Real-time Logs)
- **오류 로그** (Error Logs)
- **추적 정보** (Traces)

### 3단계: 오류 로그 찾기

1. **"관찰 가능성"** 탭에서
2. **"로그"** (Logs) 또는 **"오류"** (Errors) 섹션 확인
3. **최근 오류 메시지 확인**
4. **오류 메시지 클릭하여 상세 정보 확인**

## 🔍 대안: 메트릭 탭에서 확인

"메트릭" 탭에서도 일부 오류 정보를 확인할 수 있습니다:
- **오류율** (Error Rate)
- **오류 수** (Error Count)

## 📋 로그에서 확인할 정보

로그에서 다음 정보를 찾으세요:
- **오류 메시지**: 예외 내용
- **Traceback**: 오류 발생 위치
- **타임스탬프**: 오류 발생 시간
- **요청 정보**: 어떤 요청에서 오류가 발생했는지

## 🆘 로그가 보이지 않는 경우

### 방법 1: Workers URL 직접 테스트

1. **Workers URL을 브라우저에서 열기**
2. **F12** → **Network 탭**
3. **요청 클릭** → **Response 탭**
4. **오류 메시지 확인**

### 방법 2: 배포 탭에서 확인

1. **"배포"** (Deployments) 탭 클릭
2. **최신 배포 확인**
3. **배포 상태 확인**
   - ✅ 성공: 녹색
   - ❌ 실패: 빨간색 (오류 메시지 표시)

### 방법 3: Workers 재배포

로그가 보이지 않으면 Workers를 재배포하여 새로운 오류를 발생시킬 수 있습니다:

```powershell
cd C:\Users\malgn\Desktop\malgpt\worker
npx wrangler deploy
```

배포 후 Workers URL을 다시 테스트하면 새로운 로그가 생성됩니다.
