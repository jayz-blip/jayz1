# 로그 확인 필수 가이드

## 🚨 중요: 로그를 확인해야 합니다!

500 에러가 계속 발생하는데, **로그를 확인하지 않으면 정확한 원인을 알 수 없습니다.**

## ✅ 로그 확인 방법

### 1단계: Cloudflare 대시보드 접속

1. **https://dash.cloudflare.com/** 접속
2. 로그인

### 2단계: Workers 프로젝트로 이동

1. **Workers & Pages** 클릭
2. **Workers** 클릭
3. **`chatbot-api`** 프로젝트 클릭

### 3단계: 관찰 가능성 탭 확인

1. **"관찰 가능성"** (Observability) 탭 클릭
2. **"이벤트"** (Events) 탭 클릭

### 4단계: 오류 이벤트 확인

1. **최근 이벤트** 중 **빨간색(오류)** 표시된 이벤트 클릭
2. **오류 메시지** 확인
3. **스택 트레이스** 확인
4. **전체 오류 내용 복사**

## 📋 확인할 내용

### 필수 확인 사항

1. **오류 메시지**: 정확한 오류 메시지가 무엇인가?
2. **스택 트레이스**: 어느 함수에서 오류가 발생했는가?
3. **라인 번호**: 어느 라인에서 오류가 발생했는가?
4. **타임스탬프**: 언제 오류가 발생했는가?

### 예시 오류 메시지

```
TypeError: Cannot read property 'run' of undefined
→ AI Workers 바인딩 문제

Error: D1 database binding is not configured
→ D1 바인딩 문제

SyntaxError: Unexpected token
→ 코드 문법 오류
```

## 🔍 로그 확인 후 할 일

1. **오류 메시지 복사**
2. **스크린샷 찍기** (가능하면)
3. **오류 메시지를 개발자에게 전달**

## 💡 팁

1. **로그가 가장 중요합니다!**
   - 로그 없이는 정확한 원인을 알 수 없습니다
   - 반드시 로그를 확인하세요

2. **최근 이벤트 확인**:
   - 가장 최근에 발생한 오류를 확인하세요
   - 여러 오류가 있으면 가장 최근 것을 확인하세요

3. **오류 메시지 전체 복사**:
   - 일부만 보면 안 됩니다
   - 전체 오류 메시지와 스택 트레이스를 복사하세요
